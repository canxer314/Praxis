# Where does AgentOS V3 sit?

## 架构定位

与 V2 相同：AgentOS V3 运行在 **OpenClaw 内部**（作为 memory plugin），通过 MCP 连接 **AgentMemory**（存储后端）。核心定位不变。

V3 的变化不在"位置"，而在"内部结构"——新增了 Curiosity Engine 和四维能力模型。

## V2 → V3 架构变化

| 维度 | V2 | V3 |
|------|----|----|
| AgentOS 位置 | OpenClaw Memory Plugin | 不变 |
| 工具执行 | OpenClaw 原生，AgentOS 观察 | 不变 |
| 核心 Hook | 5 个（全工具生命周期） | 6 个（+message_received） |
| 能力模型 | `{tool → proficiency}` | `{tool + domain + task + user → proficiency/familiarity/confidence}` |
| 知识分类 | 无结构化分类 | 五类：领域/任务模式/用户偏好/流程/工具 |
| 学习引擎 | 被动响应 Hook | 被动 + Curiosity Engine 主动检测 |
| 用户交互方向 | 单向（用户→AI） | 双向（用户⇄AI） |

## 与各系统的关系（与 V2 一致，除以下更新）

### AgentMemory — 新增数据类型

V3 向 AgentMemory 写入新的数据类型（详见 [design.md](design.md)）：

- `DomainKnowledge`：领域概念、干系人、项目信息
- `TaskPattern`：任务结构、工具链、常见陷阱
- `UserModel`：沟通偏好、决策模式、优先级信号
- `KnowledgeGap`：已识别但未填补的知识缺口

### OpenClaw — 新增 Hook 使用

V3 新增使用 `message_received` hook 捕获非工具学习信号。同时可能使用 `message_sending` hook 在 AgentOS 主动提问时注入消息。
