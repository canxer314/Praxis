# What is Praxis V5?

> V5 = V4 的过程模型 + **Meta Layer** = 一个能发现自己"不会想"然后学会"怎么想"的 Praxis

## 一句话定义

**Praxis V5 在 V4 的"过程驱动引擎"之上，增加了一个元认知层（Meta Layer）——它不处理具体的任务、知识、流程，它观察 Praxis 自身的认知过程，发现"我现有的思考框架不够用"，然后构建新的认知结构来应对——像一个真正的人一样，不是等着别人告诉自己该怎么想，而是自己发现"我不会想"，然后自己学会怎么想。人类从设计者变为治理者。**

---

## Is / Is-not

| Is | Is-not |
|----|--------|
| 观察自身认知过程并发现结构性不足的元层 | 另一个处理具体任务的执行层 |
| 在人类治理下构建新认知结构的自主系统 | 不受约束地自我修改的黑盒 |
| 从"模板匹配失败"中检测结构缺口 | 从零先验场景中构建认知（那是 V6） |
| 人类审核 + Praxis 实验的协作进化 | 完全自主的 AGI（人类仍在回路中） |
| 与 V4 的 Process Engine 协作（Process Engine 处理已知流程，Meta Layer 处理"Process Engine 不够用"的情况） | Process Engine 的替代品 |

## V1 → V2 → V3 → V4 → V5 演进

| 版本 | 核心命题 | Praxis 能做到的新事 |
|------|---------|-------------------|
| V1 | AI 需要跨会话持久成长 | 概念建立 |
| V2 | 找到正确的载体 | 在 OpenClaw 内部运行 |
| V3 | 学什么、谁发起 | 主动发现知识缺口并提问 |
| V4 | 下一步该干什么 | 推动多角色流程、处理阻塞 |
| **V5** | **当我不知道怎么干的时候，我该怎么想** | **发现认知框架不足，构建新框架，在人类审核下实验固化** |

## 场景：谈判——V4 的边界和 V5 的突破

### V4 能处理的

> 用户："帮我跟供应商谈判这个合同的价格"
>
> V4：匹配 ProcessTemplate → 最接近的是"合同审核流程"（适配度 0.2）
> → 强行套用：步骤1 发给供应商 → 步骤2 等待回复 → 步骤3 确认/修改
> → 供应商说"价格不能降，但我们可以加服务"
> → Process Engine 无法处理这种"不是接受/拒绝，而是利益交换"的非二元响应
> → 卡住 → 升级给用户 → **用户手动处理**

### V5 能处理的

> 用户："帮我跟供应商谈判这个合同的价格"
>
> V4 尝试匹配 → 最佳适配度 0.2 → **触发 Meta Layer**
>
> **Structural Inadequacy Detector**：
> - 检测到这不是单次失败——是结构性不匹配
> - 合同审核流程假设"确认或修改"，谈判是"利益交换"
> - 聚类上 3 次类似场景（价格讨论、条款协商）→ 全部失败或需要大量用户介入
> - → 输出 StructuralGap："缺少处理'协商议价'场景的认知结构"
>
> **Cognitive Structure Constructor**：
> - 分析失败模式的共同点：线性步骤不适配、二元决策不适配、角色关系是竞合而非协作
> - 生成候选结构 NegotiationModel：
>   - 核心概念：利益映射 / 交换空间 / BATNA / 让步序列
>   - 核心流程：准备→开局→探索→交换→收束→确认
> - → 向用户提议：
>   > "我注意到谈判类场景我一直处理不好。问题在于我在用'执行任务'的方式处理'协商'。谈判不是对方确认或拒绝——它是利益交换。我设计了一种新的思考方式：先梳理双方的利益点，找到可以交换的维度，制定让步策略。要试试吗？如果效果好，我以后遇到谈判就用这个方式。如果不好，我继续调整。"
>
> 用户同意 → 试点 → 验证有效 → 固化到 Cognitive Structure Registry
> 下次谈判场景 → 自动激活 NegotiationModel

### V5 能，V4 不能

| 能力 | V4 | V5 |
|------|----|----|
| 发现"线性流程模板"不适配谈判 | ❌ | ✅ StructuralGap |
| 自己提出新认知结构 | ❌ | ✅ Cognitive Structure Constructor |
| 与用户协作实验新结构 | ❌ | ✅ StructureExperiment |
| 验证通过后固化新结构 | ❌ | ✅ Structure Registry |
| 下次自动匹配新结构 | ❌ | ✅ |
