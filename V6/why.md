# Why AgentOS V6?

> V5 需要"匹配失败的模板"来检测结构不足。但人类第一次去医院时，没有任何模板。V6 解决：在适配度为 0 的场景中，AgentOS 如何从零开始——不是匹配，而是感知。

---

## 一、V5 的隐含前提

V5 的 Structural Inadequacy Detector 依赖一个前提：**至少有一个能匹配的模板**。

```
谈判场景: 匹配到"合同审核流程" → 适配度 0.2 → 信号产生
竞品分析: 匹配到"数据采集流程" → 适配度 0.15 → 信号产生 (弱)
医院就诊: 匹配到？ → 无匹配 → 适配度 = 0
```

当适配度 = 0 时：
- 没有"匹配失败的模板"可以聚类
- 没有"类似场景的失败历史"可以参照
- Structural Inadequacy Detector 无法产生有意义的信号

V5 在适配度 > 0 的场景中工作。零先验场景是它的盲区。

---

## 二、人类在零先验场景中的三种本质认知操作

### 场景：第一次陪奶奶去医院，之前对医院毫无概念

#### 认知操作 A：开放感知（Open Perception）

```
不是: "这是什么类型的场景？"（分类——但没类别可用）
不是: "这和我经历过的什么类似？"（类比——没东西可比）
不是: "我应该怎么做？"（规划——不知道目标是什么）

而是: "我注意到 X。我注意到 Y。我注意到 X 和 Y 似乎有关系，但还不确定。"
```

本质是**标记 salient 元素而不强行理解**。保持认知的开放状态。允许"我不知道这是什么"。

#### 认知操作 B：原型构造（Proto-Structure Construction）

```
从 salient 元素的共现中形成模糊的、概率性的、可修正的假设:

"A（挂号）似乎总是在 B（等叫号）之前" → 置信度 0.4
"这个穿白衣服的人（医生）似乎负责问问题和做决定" → 置信度 0.35
"这个地方（医院）的目的似乎是找出问题和处理问题" → 置信度 0.25

关键: 这些不是"知识"——它们带置信度，随时准备被推翻。
```

本质是**从原始经验中涌现结构——而不是从预置的类别中匹配结构**。

#### 认知操作 C：迭代精化（Iterative Refinement）

```
每一次新互动都在更新原型:

第 2 次去医院: "挂号→等叫号" 置信度 0.4 → 0.7
第 3 次去医院: "挂号→等叫号→被问→检查→再问→开药" 置信度 0.6
第 5 次去医院: 序列置信度 → 0.9，可以固化为"流程"
第 6 次去医院: 发现"急诊和门诊完全不一样" → 出现了一个新的概念维度！
```

本质是**原型通过持续互动被修正、精化、最终固化——但在任何时候都可以被新经验推翻**。

## 三、为什么 Proto-Cognitive Engine 是必要的

V5 的 Meta Layer 处理**已知未知**（known unknowns）——"我知道谈判类场景我处理不好"。

V6 的 Proto-Cognitive Engine 处理**未知未知**（unknown unknowns）——"我甚至不知道有'医院就诊'这种类型的场景存在"。

```
V3 Curiosity:      KnowledgeGap → "我不懂 X"
V5 Meta Layer:     StructuralGap → "我需要新的认知框架来处理 Y 类场景"
V6 Proto-Cognitive: ZeroPriorGap → "我甚至不知道这是什么类型的场景——让我先观察"
```

---

## 四、为什么层定义也会被修改

如果 AgentOS 真的能从零先验场景中构建新认知结构，那它可能在某个场景中发现：

- "这个场景更适合用网络结构而不是层级结构来组织认知"
- "L2 和 L4 之间的边界在这个场景中模糊了——它们处理的东西本质上是连续的"
- "Meta Layer 的 Structural Inadequacy Detector 在零先验场景中有盲区——我需要一个新的检测维度"

这些都是**对架构本身的修改——不只是增加新结构，而是修改结构的组织方式**。

V5 不允许这种修改（Meta Layer 观察 L1-L6，但不重新定义它们）。V6 允许——在人类治理下。

---

## 五、V5 → V6 变化

| 维度 | V5 | V6 |
|------|----|----|
| **能处理的场景** | 适配度 > 0（有模板可"部分匹配"） | **适配度 = 0（零先验）** |
| **检测方式** | 模板匹配失败 → StructuralGap | **开放感知 → 标记 salient 元素** |
| **构造方式** | 从失败根因归纳 → 生成结构 | **从原始共现 → 模糊原型 → 互动精化** |
| **信息存在形式** | 确定的 CognitiveStructure | **概率性 ProtoStructure（置信度 0.2-0.9）** |
| **能修改什么** | 增加/修改 CognitiveStructure | **修改层定义、合并层、替代层级结构** |
| **人类角色** | 审核新结构提案 | **递归终止条件 + 架构变更的最终审批** |

## 六、反向论证

**反方论点**："零先验场景在现实中极其罕见。大多数'新场景'都有某种程度的可类比性——去医院至少和'去办事大厅'有类似之处。V5 的模糊匹配已经够用了。"

**为什么这个论点不够**：

1. **类比的前提是"知道可以去类比"** — 一个从未见过任何类似场景的系统，不知道"医院"可以类比"办事大厅"。类比能力本身需要经验基础。
2. **AgentOS 的长期使用必然遇到零先验** — 用户的生活和工作领域是不断扩展的。从"秘书"到"程序员"到"谈判"到"心理咨询"到"陪奶奶去医院"——必然会有适配度为 0 的场景。
3. **零先验处理是通用智能的基础** — 如果不要求完全自主，V6 的人机协作模式（AgentOS 观察→提问→人类教→原型精化）比 V5 的"人类从零设计模板"更高效。AgentOS 做它可以做的（观察、标记、形成假设），人类做人类更擅长的（解释概念、验证假设）。

**什么条件会改变结论？**
- AgentOS 的使用范围固定在少数已知领域 → V4 或 V5 足够
- 底层 LLM 不具备从原始交互中形成模糊原型的归纳能力 → V6 在工程上不可行

⚠️ 此处存在不确定性最大：LLM 能否从 raw interaction traces 中归纳出 ProtoSequence？这需要实验验证。
