# What is AgentOS V6?

> V6 = V5 的 Meta Layer + **Proto-Cognitive Engine** = 一个在完全陌生的场景中从零构建认知的 AgentOS

## 一句话定义

**AgentOS V6 在 V5 的"发现框架不够用然后构建新框架"之上，增加了一个更底层的能力：在完全陌生的场景中——没有模板可匹配、没有角色可识别、没有概念可分类、连"理解"都无从谈起时——仍能行动：先观察，标记 salient 元素，形成带置信度的模糊原型，通过与世界互动迭代精化，最终把原型固化为可复用的认知结构。这是从"匹配失败"到"零先验"的跨越。**

---

## Is / Is-not

| Is | Is-not |
|----|--------|
| 在零先验场景中从零构造认知结构 | 在已有模板匹配失败时构造新模板（那是 V5） |
| 处理"没有任何东西可以匹配"的情景 | 处理"匹配了但适配度很低"的情景（那是 V5） |
| 原型→固化结构的演化管道 | 直接生成固化结构 |
| 概率性、可修正的认知（"A 似乎总在 B 之前"，置信度 0.4） | 确定性认知（"步骤 1 之后就是步骤 2"） |
| 能修改架构的任何部分——包括层定义、包括 Meta Layer | 只能在现有层内增加结构（那是 V5） |

---

## V5 → V6 演进

| 维度 | V5 | V6 |
|------|----|----|
| **场景前提** | 至少有一个能匹配的模板（哪怕适配度 0.2） | **零先验——没有任何模板、角色、分类可用** |
| **检测方式** | 模板匹配失败 + 适配度低 | **开放感知——标记 salient 元素但不强行分类** |
| **构造方式** | 从失败模式中归纳结构缺口 → 生成新结构 | **从原始共现模式中形成模糊原型 → 互动精化** |
| **知识形式** | 确定的结构（新 CognitiveStructure） | **模糊的原型（ProtoSequence 置信度 0.3）→ 逐步精化** |
| **架构自修改** | 增加/修改 CognitiveStructure | **修改层定义、合并层、甚至替代层级结构本身** |
| **人类角色** | 审核新结构提案 | **递归终止条件（Meta 修改 Meta 时 → 必须人类验证）** |

## 场景：第一次去医院——V5 的边界和 V6 的突破

### V5 能做的

> 用户："陪我去医院，奶奶不舒服"
>
> V5：尝试匹配 ProcessTemplate → 无匹配（没有"医院就诊"模板）
> → 尝试匹配任何 CognitiveStructure → 无匹配
> → **适配度 = 0** → 没有模板可"部分匹配"
> → Meta Layer 的 Structural Inadequacy Detector 依赖**"适配度低"**作为信号
> → 当适配度 = 0（而非 0.2）时，检测器无法聚类（因为没有类似的失败可聚类）
> → V5 卡住：**"我完全不知道这是什么"**

V5 需要"匹配失败的模板"作为锚点。零先验场景没有锚点。

### V6 能做的

> 用户："陪我去医院，奶奶不舒服"
>
> AgentOS："这是我完全不了解的场景。我会先观察和学习，而不是强行套用已有框架。"

```
Phase 1: 开放感知
─────────────────
AgentOS 收集原始信息但不强行理解:
  • 位置: GPS → "医院" (这是什么东西？暂标记为地点)
  • 用户说: "要挂号" → 标记 SalientElement: "挂号" (这是什么行为？未知)
  • 用户说: "护士让我先去那边排队" → 标记 SalientElement: "排队" + "护士"
  • 用户说: "医生说要拍片子" → 标记 SalientElement: "拍片子" + "医生"
  • 外部搜索: 快速搜索"医院就诊流程概述"
  
  不试图分类。只是标记和记录。

Phase 2: 原型构造
─────────────────
检测共现模式:
  • "挂号" 总是在所有其他事情之前 → ProtoSequence: [挂号 → ???]
  • "等叫号" 在"挂号"之后反复出现 → 置信度 +0.2
  • "被问问题" 的人（"医生"）和"做检查"的人（"护士"）似乎是不同的角色
    → ProtoRole["医生"]: "问问题+做判断的人" (置信度 0.3)
    → ProtoRole["护士"]: "执行操作+引导流程的人" (置信度 0.3)
  • 整体目的似乎是"找出问题+处理问题"
    → ProtoPurpose: "诊断+治疗" (置信度 0.25)

Phase 3: 互动验证
─────────────────
AgentOS 主动提问:
  "我注意到挂号之后有一段等待，然后有人问奶奶哪里不舒服。
   这是固定的流程吗？还是每次不一样？"
  
  用户确认 → 置信度 +0.2
  
  "护士和医生有什么区别？我注意到问问题的是医生，做检查的是护士。"
  
  用户解释 → ProtoRole 精化

Phase 4: 固化决策
─────────────────
多次去医院后:
  ProtoSequence 关键步骤置信度 > 0.8
  → 候选固化: ProcessTemplate "医院就诊流程"
  → 向用户提议固化
  
  ProtoRole 置信度 > 0.8
  → 候选固化: Role 条目 "医生" + "护士"（通用角色）
```

### V6 和 V5 在同一个场景中的差异

```
V5 遇到医院场景:
  → 模板匹配: 无匹配 (适配度 = 0)
  → Structural Inadequacy Detector: "没有可参照的失败模式"（因为从没处理过类似场景）
  → 无法启动 Constructor（因为不知道从哪开始构造）
  → 结论: "我无法处理——需要人类先给我一个初始模板"

V6 遇到医院场景:
  → 声明零先验状态
  → 开放感知: 收集 salient 元素但不分类
  → 原型构造: 从共现中形成带置信度的模糊原型
  → 互动验证: 主动提问精化原型
  → 原型置信度达标 → 候选固化
  → 结论: 从零构建了"医院就诊"的认知结构
```
