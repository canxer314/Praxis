# When does AgentOS operate?

## 会话生命周期

AgentOS 在 AI 的生命周期中有明确的触发点和节奏：

```
┌────────────────────────────────────────────────────────────┐
│                     Session Lifecycle                       │
│                                                             │
│  START ────────────────────────────────────────────── END   │
│    │                                                    │   │
│    ├─ [1] 加载能力模型                                  │   │
│    ├─ [2] 恢复思维状态                                  │   │
│    ├─ [3] 注入行为指引                                  │   │
│    │                                                    │   │
│    │   ┌─── WORK PHASE ───────────────────────────┐    │   │
│    │   │                                           │    │   │
│    │   │  [4] 用户分配任务                         │    │   │
│    │   │  [5] AI 执行任务                          │    │   │
│    │   │       • 查询能力模型 → 判断自主性         │    │   │
│    │   │       • 查询相关知识 → 应用已知最佳实践   │    │   │
│    │   │  [6] 任务完成                             │    │   │
│    │   │       • AI 自我评估                       │    │   │
│    │   │       • 用户反馈                          │    │   │
│    │   │  [7] 实时学习事件触发                     │    │   │
│    │   │       • 提取教训                          │    │   │
│    │   │       • 更新能力模型                      │    │   │
│    │   │  [8] 重复 4-7（多个任务）                 │    │   │
│    │   └───────────────────────────────────────────┘    │   │
│    │                                                    │   │
│    ├─ [9] 会话反射                                      │   │
│    │       • 总结本会话的关键成长                       │   │
│    │       • 检测新模式出现                            │   │
│    │       • 生成演化提案（如有需要）                   │   │
│    ├─ [10] 保存思维状态                                │   │
│    └─ [11] 能力模型版本快照                            │   │
└────────────────────────────────────────────────────────────┘
```

## 触发点详解

### 1. 会话开始时 (SessionStart Hook)

| 动作 | 频率 | 延迟敏感 | 说明 |
|------|------|---------|------|
| 加载能力模型摘要 | 每次会话开始 | 是 | 从 AgentMemory slot 获取当前 CompetencyModel，取其摘要注入系统提示 |
| 恢复思维状态 | 每次会话开始 | 是 | 检索最近的 mental_state 记忆，重建"上次停在哪里"的上下文 |
| 注入工具熟练度指引 | 每次会话开始 | 是 | 对每个已注册工具，注入简短的自主性策略（"咖啡机：自主冲，改参数需确认"）|

### 2. 任务执行期间 (During Task)

| 触发条件 | 动作 | 说明 |
|---------|------|------|
| AI 准备调用工具 | 查询该工具的 autonomy 策略 | 决定是否需要用户确认 |
| AI 需要特定知识 | 查询知识库 | 检索与该工具/任务相关的已知最佳实践 |
| AI 不确定该怎么做 | 查询该工具的 known_failure_modes | 检查是否出现过类似场景，规避已知错误 |

### 3. 任务完成后 (Post-Task)

这是核心学习触发点。每次任务完成时触发：

```
任务完成
    │
    ├─ AI 自我评估
    │   • 和期望相比完成度如何？
    │   • 遇到了什么困难？
    │   • 有什么可以改进的？
    │
    ├─ 用户反馈（如有）
    │   • 隐式：用户说"不错"/"不对，应该是..."
    │   • 显式：用户打分或提供结构化评审
    │
    └─ 学习事件自动生成
        • 比较实际表现 vs 能力模型的预期
        • 识别差距
        • 生成 LearningEvent
        • 更新受影响技能的能力评分
```

### 4. 特定事件触发

| 事件 | 触发条件 | 动作 |
|------|---------|------|
| **错误发生** | 工具调用失败或用户指正 | 立即记录错误模式，关联到当前上下文 |
| **重复错误** | 同一错误模式出现 2+ 次 | 提升为 anti-pattern，下次任务前主动提醒 |
| **突破** | 用户明确肯定（"这次做对了"）| 强化对应技能，可能提升熟练度 |
| **新工具接入** | 新的 MCP Server 或集成被安装 | 初始化工具注册表条目，设定初始自主性为 supervised |
| **用户偏好变化** | 用户说"以后用 X 而不是 Y" | 更新对应工具的 user_preferences 和 best_practices |

### 5. 会话结束时 (SessionEnd Hook)

| 动作 | 说明 |
|------|------|
| 会话摘要 + 自我反思 | 生成本次会话的成长总结 |
| 模式检测 | 检测新出现的稳定行为模式（复用 AgentMemory 的 `memory_patterns`）|
| 演化提案生成 | 如果有显著的能力变化，生成 SelfModelUpdateProposal |
| 保存思维状态 | 保存 mental_state 用于下次会话恢复 |
| 能力模型快照 | 如果能力模型有更新，保存新版本 |

### 6. 后台定期任务

受 AgentMemory 的 consolidation pipeline 启发：

| 周期 | 任务 | 说明 |
|------|------|------|
| 每 2 小时 | 知识固化 | 将碎片化知识点合并为主题知识条目 |
| 每次会话后 | 技能评估校准 | 基于最近任务结果校准能力评分 |
| 每天 | 演化叙事更新 | 生成"AI 这一周的变化"叙述（如启用）|
| 每周 | 能力缺口分析 | 识别停滞不前的能力，生成学习建议 |
| 每月 | 能力模型审计 | 检查能力评分的准确性，清理过期数据 |

## 自主性级别变化时机

```
novice ──────▶ advanced_beginner ──────▶ competent ──────▶ proficient ──────▶ expert
  │                  │                      │                  │                │
  │ 首次使用工具      │ 完成 5+ 任务          │ 完成 20+ 任务     │ 完成 50+ 任务   │ 100+ 任务
  │ 每步都需确认      │ 基础操作自主          │ 标准任务自主      │ 复杂任务自主     │ 可以教别人
  │                  │ 新参数需确认          │ 重大变化确认      │ 仅异常情况确认   │
```
