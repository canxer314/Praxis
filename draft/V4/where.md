# Where does Praxis V4 sit?

## 架构定位

与 V2/V3 相同：Praxis V4 运行在 **OpenClaw 内部**（作为 Memory Plugin），通过 MCP 连接 **AgentMemory**（存储后端）。通过 OpenClaw 的多通道能力触达用户和协作者。

V4 的最大变化不在"位置"，而在"内部结构的复杂度"——新增了四个子系统，让 Praxis 从"学习引擎"升级为"学习 + 过程驱动引擎"。

四个 V4 子系统的位置逻辑：

| 子系统 | 所在层 | 原因 |
|--------|-------|------|
| **Process Engine** | L2（任务编排） | 它是任务模型的扩展——从工具链变为过程网络 |
| **Role Model** | L3（知识管理） | 角色信息是领域知识的一个特殊类别——关于"谁"的知识 |
| **Momentum Engine** | L4（学习闭环） | 它与 Curiosity Engine 对称——前者管理"知识缺口"，后者管理"流程阻塞" |
| **Action Verification** | L4（学习闭环） | 它是 LearningEvent 体系的扩展——新增"行动正确性"这一事件类型 |

## V3 → V4 架构变化

| 维度 | V3 | V4 |
|------|----|----|
| Praxis 位置 | OpenClaw Memory Plugin | 不变 |
| 任务模型 | TaskPattern（工具链） | **ProcessTemplate（过程网络：自己做 + 找别人做 + 等待 + 推动）** |
| 角色认知 | UserModel（单人） | **RoleRegistry（多人关系网）** |
| 学习引擎 | LearningEngine + Curiosity Engine | + **Momentum Engine + Action Verification** |
| Hook 数量 | 6 个 | **7 个**（+ cron_tick 定期扫描阻塞） |
| 能力维度 | 4 维 | **6 维**（+过程管理 + 行动可靠性） |
| 用户交互 | 被动响应 + 主动提问 | + **主动推动（催办/升级/提醒）** |
