# Where does Praxis V5 sit?

## 架构定位（V5 核心变化：Meta Layer）

V5 在 V4 的架构之上增加了一个 **Meta Layer（元认知层）**。这不是在 L6 之上加一个 L7——Meta Layer 与 L1-L6 的关系不同。它是**侧面的观察者**，不是上层的指挥者。

## Meta Layer 和 L1-L6 的关系

不是层级关系——是观察关系：

```
Meta Layer
    │
    │ 观察各层的适配质量
    │
    ├──▶ L6 自主决策层: 自主性判断是否在特定场景中反复错误？
    │
    ├──▶ L5 能力模型: 哪个维度长期不增长？（不只是数值低，是"结构上无法增长"）
    │
    ├──▶ L4 学习闭环: 
    │       • Curiosity Engine 检测到的 gap 中，哪些反复出现但无法被用户回答填补？
    │         → 可能不是 KnowledgeGap，是 StructuralGap
    │       • Action Verification 中，哪些类型的验证长期低分？
    │
    ├──▶ L3 知识管理:
    │       • 知识分类方式是否阻碍了特定类型的知识组织？
    │       • Role Model 中的角色画像是否缺少对某类角色的描述维度？
    │
    ├──▶ L2 任务编排:
    │       • Process Engine 的步骤门控是否在某些场景中不够用？
    │       • ProcessTemplate 适配度是否在某些场景中持续低下？
    │
    └──▶ L1 工具熟练度: (Meta Layer 很少直接介入这一层)
```

## V4 → V5 架构变化

| 维度 | V4 | V5 |
|------|----|----|
| 架构组织 | 固定 6 层 + 4 子系统 | 固定 6 层 + 4 子系统 + **Meta Layer** + **动态认知结构库** |
| 认知结构数量 | 4（固定，人类设计） | **N（动态增长，Praxis 提议 + 人类审核）** |
| 结构能否修改 | 参数可以调，结构不能改 | **参数可以调，结构可以改（人类治理下）** |
| 谁观察架构 | 人类 | **Meta Layer + 人类** |
| 谁提出新结构 | 人类 | **Cognitive Structure Constructor + 人类审核** |
